# 会话与上下文管理

## 按任务拆分对话

不要把所有问题塞在一个对话里。

- 对话 A：产品方案
- 对话 B：代码实现
- 对话 C：文案润色

拆开能减少模型上下文混乱。

## 用项目归类对话

可以创建项目，把相关的对话放在同一个项目下。项目支持写说明和指令，项目内的对话会继承这些上下文。

## 收藏

在 AI 回复的更多菜单里可以收藏消息，方便后续查找。

## 对话分支

可以从任意一条消息创建分支，试验不同的追问方向而不影响原对话。

## 长对话的建议

1. 每轮只提一个目标，别在一条消息里塞太多要求。
2. 重要约束重复一次，比如"必须输出 JSON"。
3. 对话太长时新建一个，在第一条消息里粘贴简要背景。

## 提示词结构

按这个顺序组织输入效果比较好：

1. 背景（你在做什么）
2. 目标（你要什么输出）
3. 约束（格式、长度、风格）
4. 示例（可选）

## 本地数据与隐私

BotHub 默认把数据存在本地。登录账号后，对话和配置（包括 API Key）会同步到云端。
调用在线模型时，消息内容会发送给对应的供应商。
处理敏感数据前，先确认供应商的数据政策。
