# 供应商与模型配置

## 核心字段说明

| 字段 | 用途 | 常见错误 |
| --- | --- | --- |
| `Base URL` | 供应商接口地址 | 写错协议或路径 |
| `API Key` | 鉴权凭证 | 复制不完整、过期 |
| `Model ID` | 指定调用模型 | 名称拼写错误 |

## 推荐配置策略

1. 至少配置两个模型（主用 + 备用）。
2. 将同一供应商的不同模型分开保存，便于快速切换。
3. 按用途命名模型，例如“写作-长文”“代码-快速”。

## 排查顺序（从快到慢）

1. 检查 API Key 是否仍有效。
2. 检查 Model ID 是否与供应商后台一致。
3. 检查 Base URL 是否带多余路径。
4. 检查网络代理或公司网关是否拦截请求。

## 参数建议（通用）

- 温度（Temperature）：
  - 事实问答：偏低
  - 创作发散：偏高
- 最大输出长度（Max Tokens）：
  - 太小会截断
  - 太大可能增加成本和时延

## 安全建议

- 不要把 API Key 写入公开仓库
- 定期轮换 Key，尤其是多人协作时
- 发现泄露后立即禁用旧 Key 并重建
