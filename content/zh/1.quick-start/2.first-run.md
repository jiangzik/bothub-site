# 首次启动与第一条对话

## 目标

完成一次完整链路：

1. 配置供应商
2. 选择模型
3. 发出消息并获得回复

## 步骤 1：准备 API Key

在你选择的模型供应商后台创建 API Key，并妥善保存。  
建议单独为 BotHub 创建一组 Key，便于后续轮换和权限管理。

## 步骤 2：添加供应商

1. 打开 BotHub 设置页。
2. 找到“供应商”或“模型配置”入口（名称可能因版本略有差异）。
3. 填写以下字段：
   - `Base URL`（如供应商要求）
   - `API Key`
   - 模型名称（Model ID）
4. 保存配置。

## 步骤 3：选择默认模型

在会话页或模型选择区，选择刚配置的模型作为当前会话模型。

## 步骤 4：发送测试消息

输入一条简单消息，例如：`你好，请用一句话介绍你自己。`  
收到正常文本回复，即表示首条对话完成。

## 成功判定

- 无认证错误（401/403）
- 无模型不存在错误（404 或模型名错误）
- 响应时间可接受，回复内容正常

## 首次配置后建议

- 再增加一个备用模型，避免单一供应商不可用
- 记录你可用的模型名，减少拼写错误
- 保留一条“测试会话”，用于后续升级后回归验证
